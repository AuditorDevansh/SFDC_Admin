Security Model — Quick Reference
Layer 1 — OrgWho can log in → User licenses, IP restrictions, Login Hours
Layer 2 — ObjectWhat objects can be accessed → Profiles & Permission Sets (CRUD)
Layer 3 — FieldWhich fields are visible/editable → Field-Level Security (FLS)
Layer 4 — RecordWhich records can be seen → OWD + Roles + Sharing Rules
OWD = FloorSet to most restrictive. Open with Roles, Sharing Rules, Manual Sharing.
Profile vs RoleProfile = what you CAN DO. Role = what records you CAN SEE.
Automation Decision Tree
Default ChoiceFlow Builder — use for ALL new automation from 2023 onward
Record triggeredRecord-Triggered Flow (on create/update/delete)
User-facing wizardScreen Flow — presents UI to user
Time-basedScheduled Flow — runs at specific intervals
Human approval neededApproval Process (not a Flow)
Legacy / AVOIDWorkflow Rules and Process Builder — being retired
Report Types Compared
Tabular
No grouping. Simple list. Cannot power dashboards (unless row limit set).
Summary
Row groupings only. Subtotals. Most common. Powers dashboards.
Matrix
Row AND column groupings. Cross-dimensional. Powers dashboards.
Joined
Multiple blocks, different object types side by side.
Relationship Quick Reference
Master-DetailStrong. Cascade delete. Enables Roll-Up Summary. Child inherits parent sharing.
LookupLoose. No cascade delete. No Roll-Up Summary. Independent sharing.
Many-to-ManyUse a Junction Object with two Master-Detail relationships.
HierarchicalSpecial Lookup on User object only. For manager relationships.
Formula Logic — Key Rules
Validation = TRUEFormula TRUE = ERROR shown, save blocked. Formula FALSE = save ok.
ISBLANK vs ISNULLUse ISBLANK for text fields. ISNULL deprecated — use ISBLANK instead.
Cross-object formulaUse dot notation: Account__r.Name — max 10 hops per formula.
Roll-Up SummaryOnly on Master side of Master-Detail. Operations: COUNT, SUM, MIN, MAX.
